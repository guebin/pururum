## Phase 1 — 이미지 저장 토스트 & 첫 팝업 작업

**요청**: "이미지를 저장하려면 먼저 문서를 저장하세요 이거 또나오네"
- **원인**: 제목없는 문서는 파일이 없어 `attachments/`에 그림 저장 불가.
- **해결**: `ensureDocSaved()` — 낙서/붙여넣기 전 강제 자동저장. (cfa9dfb)

**요청**: "이미지.. 표처럼 별도 팝업 + 삭제도"
→ **이미지 팝업 편집기** + **오브젝트 클릭 후 Backspace 삭제** + **소스토글 동그란 버튼**
+ 표/그림 위아래 공백줄(`insertBlockText`) + 도움말 보강. (c9038c0)

**⚠️ 대형 사고**: 새 `insertBlock` 함수가 삽입 메뉴의 동명 함수를 덮어써 표·탭셋·콜아웃
삽입이 전부 생짜 텍스트를 넣음(전멸). → `insertBlockText`로 개명 + **이미지 위 낙서(캔버스)**
+ **호버 × 배지**(모든 오브젝트). (d813acc)

**요청**: "오브젝트 삽입/클릭/삭제 다 같은 로직으로 묶어 정리" + "콜아웃도 하나만 넣고
종류 선택"
→ **콜아웃 오브젝트화**(단일 삽입 + 종류 팝업 노트/팁/경고/주의/중요). (d5c22dd)

**요청**: "콜아웃 수정이 반영 안돼" + "표 × 이상한 곳" + "코드 지저분, 정리해"
- **원인(콜아웃)**: 실기 WKWebView서 팝업 포커스 핸드오프로 위젯 재렌더→`el.isConnected=false`
  →위치 못 찾아 조용히 무시(헤드리스 재현 안 됨).
- **해결**: 공용 **`findObjRange`**(DOM 위치 우선, 실패 시 문서 전체 유일매치 폴백) +
  `removeObjRange`로 표·탭셋·콜아웃·이미지 전부 통일. 죽은 코드(applyImageAttr, #ctxbar,
  구 그립/정렬바) 대거 삭제.
- **표 × 위치**: `.qv-block.qv-hastable{width:fit-content}`로 표 너비에 맞춤. (6993d2f)

## Phase 2 — 코드 오브젝트화 → 되돌림, 소스토글 ⌘E, 리스트/폰트 대전환

**요청**: "코드도 오브젝트화 (삽입엔 넣지말고 아님말고)" + 낙서 메뉴명 + 소스단축키
→ **코드 오브젝트화**(팝업), 메뉴 "그림(낙서)"→"낙서", **소스토글 ⌘/→⌘E**(⌘/는 주석 충돌). (65f988d)

**요청**: "itemize 안예쁨 / 신록예찬 스타일 맞어? / 블로그 보고 다듬어 / quarto도 있음"
- 블로그(`~/Dropbox/01-rsch/9999-Yechan`) styles.css는 리스트 미커스텀=부트스트랩 기본.
- 처음 눈대중(1.6em→0.6em)은 다 틀림. **Quarto 1.8.27로 cosmo 렌더 실측** → 본문 대비
  **레벨당 2em**. 에디터는 불릿 in-flow라 depth 계산 `padding=depth·1.4−0.66em`
  →레벨1=2.00em·레벨2=4.00em. (83bd54e, ec067b9, 2f272ce, **51997d7**)

**요청(폰트)**: "신록예찬이랑 똑같이 / 나눔명조로 되돌리기 / 글꼴 마음에 안들어"
- GitHub 산세리프 폐기 → **나눔명조**. ⚠️ `url('fonts/*.otf')`는 실기 WKWebView서 로드 실패
  →**fonts.css에 base64 임베드**. (a1542ad, **aa91bf3**)

**요청**: "출력 인라인수식 이상" → 출력본이 MathJax woff 못 찾아 이탤릭 아닌 세리프 폴백
→ `inlineFontUrls`로 폰트 base64 임베드. (a1542ad)

**요청**: "캡션 그림 바로 밑 / 크기 슬라이더 없애(드래그로) / 표 크기조정?"
→ 이미지 **드래그 리사이즈 부활** + 슬라이더 제거 + 캡션 위치. 표는 마크다운 특성상 폭 지정
없음(설명). 버튼 2개(소스·도움말) 균형. (a1542ad)

**⚠️ 폴더 rename 발견**: 세션 중 `verdure`→`pururum`으로 폴더명 바뀜(cwd deleted 에러로
발견). git·파일 무사, 원격은 그대로. rename 후 첫 빌드 실패(cargo 캐시 옛 경로)
→`cargo clean` 후 재빌드. 이후 경로 `pururum`.

## Phase 5 — guebin/pururum 배포 + 인쇄 폐기 + HTML export

**요청**: "guebin/pururum에 푸시 / README는 ink(github.com/guebin/ink#install-macos)
참고 / ink처럼 3가지 설치"
- ink README 분석: **Homebrew tap / 원커맨드 curl(hdiutil+ditto+xattr) / 수동 dmg+xattr**.
  모두 **Release의 .dmg + Homebrew cask** 필요.
- guebin/pururum 이미 존재(비어있음), 내 토큰(miruetoto)은 **push권한만(admin 아님)**.

**요청(중간)**: "인쇄는 tabset 때문에 안되니 없애자(1.0.0) / 대신 단일 html export(1.1.0)"
- 네이티브 인쇄 전면 제거(print_page/@media print/print.css).
- **출력 버튼(⌘P)=`doExport`**: 자체완결 .html — fonts.css+github+doc.css+`inlineFontUrls`
  (MathJax woff base64)+mjxCss 인라인, 이미지=data URI, 수식 pre-render, **탭셋 인라인
  스크립트로 인터랙티브**. 저장: 데스크톱 Rust `export_html`(save dialog), 웹 Blob 다운로드.
  검증 5.7MB 파일 file:// 독립 렌더. → 효율 위해 인쇄제거+export를 **한 릴리즈 1.1.0**으로.
- 도움말 제목 중복(프론트매터+H1) 제거, 버전 1.1.0. (e8c2f78)

**배포 실행**:
- `Casks/pururum.rb`(version 1.1.0, dmg sha256, postflight xattr, zap com.local.verdure).
- README ink 스타일 재작성(3설치).
- `git remote add guebin`, `git push guebin main`(신규) + `origin`.
- `gh release create v1.1.0 -R guebin/pururum` + **Pururum.dmg**(aarch64, ~7.4MB) 업로드.
  → `releases/latest/download/Pururum.dmg` 동작.
- ⚠️ **Pages 못 켬(404, admin 아님)** → guebin 계정이 Settings→Pages→"GitHub Actions" 수동.

**요청**: export 토스트 "이건뭐닝" (한글 파일명 NFD로 깨져 보임)
→ macOS는 한글 파일명 NFD 저장 → `describePath`가 `.normalize("NFC")`. (fed419d)
